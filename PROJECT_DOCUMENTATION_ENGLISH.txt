J.A.R.V.I.S AI PROJECT DOCUMENTATION
====================================

Project Name:
J.A.R.V.I.S AI

Full Form:
Just A Rather Very Intelligent System

Purpose:
This project is a local AI assistant backend. It receives user messages, generates AI responses, saves chat history, retrieves relevant context from local learning data, and supports realtime web-search based answers.

In simple terms:
A user sends a message from Chrome, FastAPI docs, or any frontend. The FastAPI backend receives the message. ChatService manages the session. GroqService calls the AI model. RealtimeGroqService adds Tavily web search for current information. VectorStoreService retrieves useful context from local memory and learning data.


WHY THIS PROJECT WAS CREATED
============================

1. To create a personal AI assistant backend.

2. The assistant is designed to:
   - Answer normal chat questions.
   - Answer realtime questions using web search.
   - Improve answers using local learning data.
   - Remember previous chat history by session.
   - Run locally as a FastAPI server.

3. The backend is reusable:
   - It can be called from Chrome.
   - It can be tested from FastAPI docs.
   - It can later be connected to a frontend, desktop app, browser extension, or mobile app.


PROJECT START FLOW
==================

Main startup flow:

1. The user runs:
   python run.py

2. run.py starts the Uvicorn server.

3. Uvicorn imports:
   app.main:app

4. app/main.py loads the FastAPI application.

5. FastAPI runs the lifespan startup function.

6. During startup, these services are initialized:
   - VectorStoreService
   - GroqService
   - RealtimeGroqService
   - ChatService

7. The server becomes available at:
   http://localhost:8000

8. API documentation becomes available at:
   http://localhost:8000/docs

9. The user sends a request to /chat or /chat/realtime.

10. ChatService creates or loads a chat session.

11. GroqService or RealtimeGroqService generates the AI response.

12. The response is returned to the user.

13. Chat history is saved to disk.


MAIN REQUEST FLOW
=================

Normal chat flow:

User
-> POST /chat
-> app.main.chat()
-> ChatService.process_message()
-> GroqService.get_response()
-> Groq API
-> AI response
-> ChatService saves the assistant message
-> FastAPI returns ChatResponse


Realtime chat flow:

User
-> POST /chat/realtime
-> app.main.chat_realtime()
-> ChatService.process_realtime_message()
-> RealtimeGroqService.get_response()
-> Tavily search
-> VectorStore context retrieval
-> Groq API
-> AI response
-> ChatService saves the assistant message
-> FastAPI returns ChatResponse

Function: lifespan(app)
-----------------------

Purpose:
Controls what happens when the FastAPI app starts and stops.

Decorator:
@asynccontextmanager

Explanation:
Code before yield runs during startup.
Code after yield runs during shutdown.


Startup steps:

1. print_title()
Clears the terminal and prints the title.

2. vector_store_service = VectorStoreService()
Creates the vector store service.

3. vector_store_service.create_vector_store()
Loads learning data and chat history, creates chunks, creates embeddings, and builds a FAISS vector store.

4. groq_service = GroqService(vector_store_service)
Creates the normal Groq AI service.

5. realtime_service = RealtimeGroqService(vector_store_service)
Creates the realtime AI service with Tavily search support.

6. chat_service = ChatService(groq_service, realtime_service)
Creates the service that manages chat sessions and message processing.

7. yield
The application starts accepting requests.


Shutdown steps:

1. After the server stops, the code after yield runs.

2. Active sessions are saved using:
   chat_service.save_chat_session(session_id)

3. A shutdown message is logged.

Function: _invoke_llm(prompt, messages, question)
-------------------------------------------------

Purpose:
Sends the final prompt to Groq and returns the model response.

Inputs:
prompt:
ChatPromptTemplate object.

messages:
Chat history in LangChain message format.

question:
Current user question.

Flow:
1. Count available Groq clients.
2. Select the starting API key using round-robin logic.
3. Increment the shared key index.
4. Try the selected key.
5. Build the chain:
   prompt | self.llms[i]
6. Call:
   chain.invoke({"history": messages, "question": question})
7. Return response.content on success.
8. If the key fails, log the failure.
9. If multiple keys exist, try the next key.
10. If all keys fail, raise the final error.

Why:
This improves reliability when keys are rate-limited or temporarily fail.


Function: get_response(question, chat_history=None)
---------------------------------------------------

Purpose:
Generates a normal chat response.

Flow:
1. Try to retrieve relevant context from the vector store.
2. Get current time and date.

4. Add relevant context to system_message if found.
5. Create a ChatPromptTemplate with:

   - history placeholder
   - current human question
6. Convert chat history into HumanMessage and AIMessage objects.
7. Call _invoke_llm().
8. Return the response.

Called by:
ChatService.process_message()

Function: search_tavily(query, num_results=5)
---------------------------------------------

Purpose:
Searches the web using Tavily and returns formatted results.

Flow:
1. If tavily_client is missing, return an empty string.
2. Use with_retry() to call tavily_client.search().
3. Search parameters:
   query: user question
   search_depth="basic"
   max_results=num_results
   include_answer=False
   include_raw_content=False
4. Read results from the response.
5. If results are empty, return an empty string.
6. Format each result with:
   Title
   Description
7. Return the formatted search results.
8. If an error occurs, log it and return an empty string.

Used by:
RealtimeGroqService.get_response()


Function: get_response(question, chat_history=None)
---------------------------------------------------

Purpose:
Generates a realtime answer using search results, local context, history, and Groq.

Flow:
1. Search Tavily:
   search_results = search_tavily(question)
2. Retrieve local context from vector store.
3. Get current time and date.
4. Create system_message using:
   JARVIS_SYSTEM_PROMPT + current time/date
5. If search_results exist, escape them and add them to system_message.
6. If local context exists, escape it and add it to system_message.
7. Create a ChatPromptTemplate with:
   - system message
   - history placeholder
   - current human question
8. Convert chat history into HumanMessage and AIMessage objects.
9. Call:
   self._invoke_llm(prompt, messages, question)
10. Return the AI response.

Called by:
ChatService.process_realtime_message()

-----------------------------------------

1. Session ID must not be empty.
2. It must not contain "..".
3. It must not contain "/".
4. It must not contain "\\".

Function: get_or_create_session(session_id=None)
------------------------------------------------

Purpose:
Returns a valid session ID and ensures the session exists in memory.

If no session_id is provided:
1. Create a new UUID.
2. Create an empty session.
3. Return the new session ID.

If session_id is provided:
1. Validate the session ID.
2. If invalid, raise ValueError.
3. If the session already exists in memory, return it.
4. Try to load it from disk.
5. If loading succeeds, return it.
6. Otherwise create a new empty session using the provided ID.

Used by:
app.main.chat()
app.main.chat_realtime()

------------------------------------------------

Purpose:
Adds a user or assistant message to a session.

Flow:
1. If the session does not exist, create it.
2. Create ChatMessage(role=role, content=content).
3. Append it to the session list.


Function: get_chat_history(session_id)
--------------------------------------

Purpose:
Returns all messages for a session.

Return:
List[ChatMessage]

If session does not exist:
Returns an empty list.


Function: format_history_for_llm(session_id, exclude_last=False)
----------------------------------------------------------------

Purpose:
Formats chat history for the LLM prompt.

Why:
The AI model needs previous messages in user-assistant pairs.

Flow:
1. Get all messages for the session.
2. If exclude_last=True, skip the last message.
   This is useful because the latest user message is passed separately as the current question.
3. Loop through messages.
4. Convert complete user-assistant pairs into tuples:
   (user_message, assistant_message)
5. Ignore incomplete or mismatched pairs.
6. Keep only the latest MAX_CHAT_HISTORY_TURNS pairs.
7. Return the formatted history list.


Function: process_message(session_id, user_message)
---------------------------------------------------

Purpose:
Processes one normal chat message.

Flow:
1. Add the user's message to the session.
2. Format previous history for the AI model.
3. Call:
   groq_service.get_response(question=user_message, chat_history=chat_history)
4. Add the assistant response to the session.
5. Return the response.

Called by:
app.main.chat()


Function: process_realtime_message(session_id, user_message)
------------------------------------------------------------

Purpose:
Processes one realtime chat message.

Flow:
1. Check whether realtime_service exists.
2. Add the user's message to the session.
3. Format previous history for the AI model.
4. Call:
   realtime_service.get_response(question=user_message, chat_history=chat_history)
5. Add the assistant response to the session.
6. Return the response.

Called by:
app.main.chat_realtime()


Function: save_chat_session(session_id)
---------------------------------------

Purpose:
Saves a session's messages to a JSON file.

Flow:
1. If the session is missing or empty, do nothing.
2. Convert session_id into a safe file name.
3. Build filename:
   chat_{safe_session_id}.json
4. Create a dictionary with:
   - session_id
   - messages
5. Write the dictionary to JSON.
6. If writing fails, log an error.

Important note:
The current code uses open(filepath, "W", ...). Python file mode should normally be lowercase "w". If saving fails, change "W" to "w".


FILE: app/utils/time_info.py
============================

Purpose:
Generates current date and time information for the AI prompt.


Function: get_time_information()
--------------------------------

Purpose:
Returns a formatted string containing the current day, date, month, year, and time.

Flow:
1. Get current datetime using datetime.datetime.now().
2. Format day, date, month, year, hour, minute, and second.
3. Return the formatted string.

Used by:
GroqService.get_response()
RealtimeGroqService.get_response()

Why:
The assistant receives current time context before answering.


FILE: app/utils/retry.py
========================

Purpose:
Provides retry logic for temporary API or network failures.


Function: with_retry(fn, max_retries=3, initial_delay=1.0)
----------------------------------------------------------

Purpose:
Runs a function with retry behavior.

Inputs:
fn:
The function to run.

max_retries:
Maximum number of attempts.

initial_delay:
Wait time before the first retry.

Flow:
1. Set delay to initial_delay.
2. Run fn().
3. If it succeeds, return the result.
4. If it fails and attempts remain:
   - Log a warning.
   - Sleep for delay seconds.
   - Double the delay.
5. If the final attempt fails, raise the error.

Used by:
RealtimeGroqService.search_tavily()

Important note:
The current code sets last_exception = 0 inside the except block. A better version would use:
last_exception = e

DATA FLOW EXPLANATION
=====================

User request JSON:

{
  "message": "Hello",
  "session_id": "optional-session-id"
}

Step 1:
FastAPI validates this request using ChatRequest.

Step 2:
ChatService creates or loads a session.

Step 3:
VectorStoreService finds relevant local context.

Step 4:
GroqService or RealtimeGroqService creates the final prompt.

Step 5:
Groq API generates the answer.

Step 6:
ChatService stores the assistant message.

Step 7:
FastAPI returns:

{
  "response": "AI answer here",
  "session_id": "same-or-new-session-id"
}

COMMON ERRORS AND MEANING
=========================

Error:
cannot import name 'ChatResponse'

Meaning:
ChatResponse class was missing in app/models.py.

Fix:
Add the ChatResponse class.


Error:
cannot import name 'LERNING_DATA_DIR'

Meaning:
There was a spelling mismatch. config.py defines LEARNING_DATA_DIR.

Fix:
Use LEARNING_DATA_DIR in vector_store.py.


Error:
cannot access local variable 'system_message'

Meaning:
system_message was used before it was created.

Fix:
Create system_message before appending search results or context.


Error:
TAVILY_API_KEY not set

Meaning:
Realtime search API key is missing.

Fix:
Add TAVILY_API_KEY to .env.


Error:
No Groq API keys configured

Meaning:
GROQ_API_KEY is missing.

Fix:
Add GROQ_API_KEY to .env.

SHORT SUMMARY
=============

This project is a FastAPI-based AI assistant backend. run.py starts the server. app/main.py defines the API and initializes services. ChatService manages sessions and chat history. GroqService generates normal AI responses. RealtimeGroqService adds Tavily search for realtime answers. VectorStoreService builds local memory using FAISS. config.py manages API keys, paths, models, settings, and the assistant system prompt.

End-to-end flow:
User message
-> FastAPI route
-> ChatService
-> GroqService or RealtimeGroqService
-> Vector context and optional web search
-> Groq AI response
-> Save chat history
-> Return response to user
