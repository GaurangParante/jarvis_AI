J.A.R.V.I.S AI PROJECT DOCUMENTATION
====================================

Project Name:
J.A.R.V.I.S AI

Full Form:
Just A Rather Very Intelligent System

Purpose:
Ye project ek local AI assistant backend hai. Iska kaam hai user ke message ko receive karna, us message ke liye AI response generate karna, chat history save karna, learning data se relevant context nikalna, aur realtime query ke liye web search use karna.

Simple meaning:
User Chrome ya API docs me message bhejta hai. FastAPI backend message receive karta hai. ChatService session manage karta hai. GroqService AI model ko call karta hai. RealtimeGroqService realtime search ke liye Tavily use karta hai. VectorStoreService local memory aur learning data se useful context nikalta hai.


WHY THIS PROJECT WAS CREATED
============================

1. Personal AI assistant banane ke liye.

2. Aisa assistant jo:
   - Normal chat kar sake.
   - Realtime web search kar sake.
   - User ke learning data se answer improve kar sake.
   - Previous chat history ko remember kar sake.
   - Local machine par FastAPI server ke form me run ho.

3. Is project ka backend reusable hai:
   - Chrome frontend se call ho sakta hai.
   - FastAPI docs se test ho sakta hai.
   - Kisi future UI app se connect ho sakta hai.


PROJECT START FLOW
==================

Main flow:

1. User command run karta hai:
   python run.py

2. run.py Uvicorn server start karta hai.

3. Uvicorn app.main:app import karta hai.

4. app/main.py me FastAPI app load hota hai.

5. FastAPI lifespan startup function run karta hai.

6. Startup me ye services initialize hoti hain:
   - VectorStoreService
   - GroqService
   - RealtimeGroqService
   - ChatService

7. Server ready hota hai:
   http://localhost:8000
   http://localhost:8000/docs

8. User /chat ya /chat/realtime endpoint par message bhejta hai.

9. ChatService session create/load karta hai.

10. GroqService ya RealtimeGroqService AI response generate karta hai.

11. Response user ko wapas milta hai.

12. Chat history disk par save hoti hai.


MAIN REQUEST FLOW
=================

Normal chat flow:

User -> /chat -> app.main.chat() -> ChatService.process_message()
-> GroqService.get_response() -> Groq API -> response
-> ChatService saves assistant message -> API returns ChatResponse

Realtime chat flow:

User -> /chat/realtime -> app.main.chat_realtime()
-> ChatService.process_realtime_message()
-> RealtimeGroqService.get_response()
-> Tavily search + VectorStore context + Groq API
-> response -> ChatService saves assistant message
-> API returns ChatResponse

Function: _invoke_llm(prompt, messages, question)
-------------------------------------------------

Purpose:
Final prompt ko Groq model par send karna.

Inputs:
prompt:
ChatPromptTemplate object.

messages:
Chat history messages.

question:
Current user question.

Flow:
1. Total LLM clients count n.
2. Start key round-robin index se choose.
3. shared key index increment.
4. API key masked form log.
5. For loop me keys try karta hai.
6. chain = prompt | self.llms[i]
7. chain.invoke({"history": messages, "question": question})
8. Success par response.content return.
9. Error aaye to:
   - rate limit hai to warning.
   - other error hai to warning.
   - multiple keys hain to next key try.
   - single key hai to error raise.
10. Sab keys fail ho to final exception raise.

Why:
Rate limit aur key failure handle karne ke liye.

Function: search_tavily(query, num_results=5)
---------------------------------------------

Purpose:
Tavily se realtime search results lana.

Flow:
1. Agar tavily_client nahi hai:
   warning return empty string.
2. with_retry() ke andar tavily_client.search() call.
3. Parameters:
   query: user question
   search_depth="basic"
   max_results=num_results
   include_answer=False
   include_raw_content=False
4. response se results get.
5. Agar result empty:
   warning and return empty string.
6. Results ko readable text format me convert:
   Search results for ...
   Title: ...
   Description: ...
7. formatted_results return.
8. Error aaye to log and empty string return.

Used by:
RealtimeGroqService.get_response()


Function: get_response(question, chat_history=None)
---------------------------------------------------

Purpose:
Realtime answer generate karna.

Flow:
1. Tavily search:
   search_results = search_tavily(question)
2. Vector store context retrieve:
   retriever = vector_store_service.get_retriever(k=10)
   context_docs = retriever.invoke(question)
3. Current time/date:
   time_info = get_time_information()
4. system_message create:
   JARVIS_SYSTEM_PROMPT + current time/date
5. Agar search_results present:
   Escape curly braces and add to system_message.
6. Agar vector context present:
   Escape curly braces and add to system_message.
7. ChatPromptTemplate create.
8. chat_history ko HumanMessage/AIMessage format me convert.
9. self._invoke_llm(prompt, messages, question)
10. Response return.

Called by:
ChatService.process_realtime_message()

groq_service:
Normal chat ke liye.

realtime_service:
Realtime chat ke liye.


Function: load_session_from_disk(session_id)
--------------------------------------------

Purpose:
Saved chat session ko database/chats_data se load karna.

Flow:
1. session_id ko safe filename me convert.
2. filename = chat_{safe_session_id}.json
3. File exist nahi karti to False.
4. JSON read.
5. messages list ko ChatMessage objects me convert.
6. self.sessions[session_id] = messages
7. True return.
8. Error aaye to warning and False.

Used by:
get_or_create_session()


Function: validate_session_id(session_id)
-----------------------------------------

Purpose:
Session ID safe hai ya nahi check karna.

Checks:
1. Empty nahi hona chahiye.
2. ".." nahi hona chahiye.
3. "/" nahi hona chahiye.
4. "\\" nahi hona chahiye.
5. Length 255 se zyada nahi honi chahiye.

Why:
File path attack/path traversal avoid karne ke liye.


Function: get_or_create_session(session_id=None)
------------------------------------------------

Purpose:
Session ID return karna aur ensure karna ki session memory me exist kare.

Flow if no session_id:
1. uuid.uuid4() se new ID.
2. self.sessions[new_id] = []
3. new_id return.

Flow if session_id provided:
1. validate_session_id()
2. Invalid hai to ValueError.
3. Already memory me hai to same ID return.
4. Disk se load try.
5. Agar load successful to same ID return.
6. Warna new empty session create with given ID.

Used by:
app.main.chat()
app.main.chat_realtime()


Function: add_message(session_id, role, content)
------------------------------------------------

Purpose:
Session me user/assistant message add karna.

Flow:
1. Session missing hai to empty list create.
2. ChatMessage(role=role, content=content) append.


Function: get_chat_history(session_id)
--------------------------------------

Purpose:
Session ke messages return karna.

Return:
List[ChatMessage]

If missing:
Empty list.


Function: format_history_for_llm(session_id, exclude_last=False)
----------------------------------------------------------------

Purpose:
Chat history ko LLM ke liye format karna.

Why:
LangChain prompt me history HumanMessage/AIMessage pair form me chahiye.

Flow:
1. messages = get_chat_history(session_id)
2. exclude_last True hai to last message remove.
   Ye current user message hota hai jiska answer abhi generate karna hai.
3. Loop me messages ko user-assistant pairs me convert.
4. Sirf complete pairs include.
5. MAX_CHAT_HISTORY_TURNS se zyada ho to recent turns keep.
6. List of tuples return:
   [(user_text, assistant_text), ...]


Function: process_message(session_id, user_message)
---------------------------------------------------

Purpose:
Normal chat message process karna.

Flow:
1. add_message(session_id, "user", user_message)
2. chat_history = format_history_for_llm(session_id, exclude_last=True)
3. response = groq_service.get_response(question=user_message, chat_history=chat_history)
4. add_message(session_id, "assistant", response)
5. response return.

Called by:
app.main.chat()

Purpose:

2. user message session me add.
3. history format.
4. realtime_service.get_response() call.
5. assistant response add.
6. response return.

Called by:
app.main.chat_realtime()

Function: with_retry(fn, max_retries=3, initial_delay=1.0)
----------------------------------------------------------

Purpose:
Given function ko retry logic ke saath run karna.

Inputs:
fn:
Callable function.

max_retries:
Maximum attempts.

initial_delay:
First retry se pehle wait time.

Flow:
1. delay = initial_delay
2. for attempt in range(max_retries)
3. fn() call
4. Success ho to result return.
5. Exception aaye:
   - Last attempt hai to raise.
   - Warning log.
   - time.sleep(delay)
   - delay *= 2
6. Exponential backoff retry.

Used by:
RealtimeGroqService.search_tavily()

Important note:
Current code me last_exception = 0 set ho raha hai. Better code would store actual exception:
last_exception = e

SECURITY AND PRIVACY NOTES
==========================

1. .env contains API keys, so it is ignored by Git.

2. Chat history may contain personal data, so database/chats_data should not be public.

3. learning_data may contain private information, so keep it local.

4. API allows all CORS origins right now. For production, restrict allow_origins.

5. session_id validation exists to avoid path traversal attacks.

cannot import name 'LERNING_DATA_DIR'

vector_store.py me LEARNING_DATA_DIR use.

system_message = JARVIS_SYSTEM_PROMPT + f"\n\nCurrent time and date: {time_info}"

Meaning:
Realtime search key missing hai.

Fix:
.env me TAVILY_API_KEY add.

.env me GROQ_API_KEY add.
